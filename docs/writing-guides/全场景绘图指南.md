## 核心哲学：图的本质功能

> **"一张好图，不是把观点堆进去，而是把关系揭示出来。"**

图片最重要的功能不是"展示"，而是"揭示"——它能把人类语言难以表达的**复杂多维关系**，通过视觉语言瞬间传达。判断一张图好不好的标准：

1. **密度**：它承载了多少真实信息？
2. **逻辑**：它揭示了什么结构/关系/趋势？
3. **清晰**：读者是否在 5 秒内能抓到主旨？
4. **诚实**：它是否忠实地表达了数据和关系，没有扭曲？

Edward Tufte 的核心公式：**数据墨水比（Data-Ink Ratio）= 数据墨水 ÷ 总墨水** → 最大化

### A. 数据图表（Data Charts）

> **核心判断原则**：先问「我想表达什么关系」，再选图。

#### A1. 比较类（Comparison）
*适用：不同对象、不同时间点、不同维度之间的大小比较*

| 类型 | 最佳使用场景 | 注意事项 |
|------|------------|---------|
| **柱状图（Bar Chart）** | 分类数据对比，类别 ≤12 | Y轴从0开始，避免截断 |
| **条形图（Horizontal Bar）** | 分类标签较长，排名展示 | 按数值排序更易读 |
| **分组柱状图（Grouped Bar）** | 多变量对比（≤3组）| 组内颜色一致 |
| **堆叠柱状图（Stacked Bar）** | 总量+组成双重关注 | 只有底层比较准确 |
| **100%堆叠柱状图** | 只看比例，不关心绝对值 | 不适合展示绝对量 |
| **子弹图（Bullet Chart）** | 目标vs实际进度 | Tufte 设计，高数据密度 |
| **点图（Dot Plot/Cleveland Plot）** | 替代柱状图，强调精确值 | 坐标轴需清晰 |
| **哑铃图（Dumbbell Chart）** | 两时间点的变化对比 | 视觉直观展示变化方向 |
| **斜率图（Slope Graph）** | 两节点间的变化趋势 | 适合前后对比 |
| **瀑布图（Waterfall Chart）** | 累计增减过程（如利润分解）| 强调路径 |

#### A2. 趋势类（Trend Over Time）
*适用：时间序列，关注变化轨迹*

| 类型 | 最佳使用场景 | 注意事项 |
|------|------------|---------|
| **折线图（Line Chart）** | 连续时间序列 | 时间轴等间距 |
| **面积图（Area Chart）** | 趋势+总量感知 | 多系列时易覆盖 |
| **堆叠面积图** | 总量及组成变化 | 注意视觉失真 |
| **阶梯图（Step Chart）** | 离散时间点的突变（如政策）| 强调"跳变"而非渐变 |
| **火花线（Sparkline）** | 嵌入表格的微型趋势线 | Tufte 发明，节省空间 |
| **量程图（Range Area）** | 展示置信区间/误差带 | 科研/金融常用 |

#### A3. 分布类（Distribution）
*适用：数据分布形态，关注集中趋势、离散程度、异常值*

| 类型 | 最佳使用场景 | 注意事项 |
|------|------------|---------|
| **直方图（Histogram）** | 单变量连续数据分布 | 区间宽度影响形态 |
| **箱线图（Box Plot）** | 中位数+四分位数+异常值 | 科研标准，多组比较强 |
| **小提琴图（Violin Plot）** | 分布形态更丰富（含密度）| 比箱线图信息量更大 |
| **蜂群图（Beeswarm Plot）** | 少量数据点全部展示 | 数据 <500 时清晰 |
| **密度图（Density Plot/KDE）** | 平滑版本的直方图 | 多组对比清晰 |
| **条形误差图（Bar+Error）** | 均值±SD/SEM | 科研必备，误差类型必须标明 |
| **脊形图（Ridgeline Plot）** | 多组时间序列分布对比 | 优雅，适合音乐/气候数据 |

#### A4. 关联类（Correlation & Relationship）
*适用：两变量或多变量之间的关系*

| 类型 | 最佳使用场景 | 注意事项 |
|------|------------|---------|
| **散点图（Scatter Plot）** | 两连续变量相关性 | 过绘制时用透明度 |
| **气泡图（Bubble Chart）** | 三变量（X/Y/大小）| 气泡面积而非直径编码数量 |
| **散点矩阵（Scatter Matrix）** | 多变量两两关系预览 | 探索性分析首选 |
| **相关矩阵（Correlation Heatmap）** | N×N变量相关性概览 | 颜色编码：红暖蓝冷 |
| **连接散点图（Connected Scatter）** | 两变量随时间的演化轨迹 | 需标注时间起点 |
| **六边形分箱图（Hex Bin）** | 大量散点数据的密度 | 替代散点图，避免过绘制 |

#### A5. 比例类（Part-to-Whole）
*适用：组成结构，各部分占整体的比例*

| 类型 | 最佳使用场景 | 注意事项 |
|------|------------|---------|
| **饼图（Pie Chart）** | ≤5个类别，有主导项 | 禁止3D、禁止>6片 |
| **环形图（Donut Chart）** | 同饼图，中心可加总数 | 内外环可做二级分类 |
| **旭日图（Sunburst）** | 多层级比例结构 | 读取难度随层数增加 |
| **矩形树图（Treemap）** | 大量类别的嵌套比例 | 面积编码值 |
| **马赛克图（Mosaic Plot）** | 两个分类变量的比例 | 统计可视化 |
| **华夫图（Waffle Chart）** | 直觉化的百分比（如"100人中X人"）| 传播向强，精确度低 |
| **单位图（Unit Chart/Icon Array）** | 个体化呈现，更有情感共鸣 | 医学/政策传播常用 |

#### A6. 流向类（Flow & Quantity）
*适用：量的流转、转化、路径*

| 类型 | 最佳使用场景 | 注意事项 |
|------|------------|---------|
| **桑基图（Sankey Diagram）** | 能量流/用户流/资金流转 | 流宽=量，不可交叉过多 |
| **漏斗图（Funnel Chart）** | 转化率递减（如营销漏斗）| 层间差值即流失量 |
| **弦图（Chord Diagram）** | 双向流量关系（如国家贸易）| 圆圈型，视觉震撼 |
| **冲积图（Alluvial Plot）** | 分类变量随时间的流转 | 桑基图的时间版本 |
| **流图（Stream Graph）** | 多类别趋势叠加，强调体量感 | 不精确，适合传播 |

#### A7. 多变量/高维类

| 类型 | 最佳使用场景 | 注意事项 |
|------|------------|---------|
| **热力图（Heatmap）** | 矩阵型数据，行×列的值 | 颜色映射需符合认知 |
| **雷达图（Radar/Spider）** | 多维度综合评分对比 | ≤7维，坐标轴需可比 |
| **平行坐标图（Parallel Coordinates）** | 高维数据各维度关系 | 交互版本更实用 |
| **降维散点图（PCA/t-SNE/UMAP）** | 高维数据的2D投影 | 科研专用，需说明方法 |

### B. 关系与结构图（Relationship & Structure）

> **核心目的**：揭示实体之间的关联、层级、分组。

| 类型 | 描述 | 典型场景 |
|------|------|---------|
| **思维导图（Mind Map）** | 中心节点向外发散 | 头脑风暴、笔记整理 |
| **概念图（Concept Map）** | 节点+带标签的关系箭头 | 知识结构表达（比思维导图更严谨）|
| **知识图谱（Knowledge Graph）** | 大规模实体关系网络 | AI/数据库/文献分析 |
| **树形图（Tree Diagram）** | 层级结构，从根到叶 | 分类法、决策树 |
| **组织架构图（Org Chart）** | 层级汇报关系 | 公司组织、项目团队 |
| **韦恩图（Venn Diagram）** | 集合的重叠关系 | 概念比较、特征分类 |
| **欧拉图（Euler Diagram）** | 精确的集合包含关系 | 比韦恩图更精确 |
| **网络图（Network/Graph）** | 节点+边，无明确层级 | 社交网络、引文分析 |
| **力导向图（Force-directed）** | 动态网络布局 | 展示聚类和中心节点 |
| **依赖图（Dependency Graph）** | 模块间依赖关系 | 软件工程 |
| **矩阵关系图（Matrix Diagram）** | 两组实体的对应关系 | 需求追踪、责任分配（RACI）|
| **亲和图（Affinity Diagram）** | 定性数据聚类分组 | 用户研究、设计思维 |

### D. 概念框架图（Conceptual & Explanatory）

> **核心目的**：展示抽象理论、模型、框架的内部结构与逻辑。
> **这是最能体现"逻辑感"和"有内涵"的图类**。

| 类型 | 描述 | 典型场景 |
|------|------|---------|
| **层级框架图** | 竖向/横向层次，表达从抽象到具体 | 理论模型、政策框架 |
| **矩阵框架图（2×2 Matrix）** | 两轴四象限 | BCG、艾森豪威尔矩阵、产品定位 |
| **循环图（Cycle Diagram）** | 无起点的循环过程 | PDCA、反馈回路 |
| **因果回路图（Causal Loop）** | 正负反馈回路 | 系统动力学 |
| **概念模型图** | 核心概念及其关系 | 理论框架展示 |
| **金字塔图（Pyramid）** | 层级重要性或量级 | 马斯洛需求、数据价值金字塔 |
| **漏斗/倒漏斗图** | 从广到窄或从窄到广的转化 | 认知漏斗、教育漏斗 |
| **矩阵对比表（Comparison Matrix）** | 多对象多维度打分对比 | 选型评估、竞品分析 |
| **维恩框架图** | 多概念交叉的核心区域 | 交叉领域、核心价值 |
| **同心圆图（Concentric Circles）** | 从核心向外扩展的层次 | 文化模型、生态系统 |
| **蜘蛛网图/雷达（理论版）** | 维度画像 | 能力模型、品牌特质 |
| **杠杆点图** | 干预措施的效力对比 | 政策/管理干预分析 |
| **象限/坐标系模型** | 两维度下的定性定位 | 策略选择、风格定位 |
| **接力棒模型（Relay Model）** | 串行传递过程，各阶段接力 | 知识传播、工程流水线 |

## 第二部分：通用绘图最佳实践原则

### 原则 1：先定「图的问题陈述」

**绘图之前，必须回答三个问题：**
1. **谁看？**（科学家/投资人/普通用户/社交媒体用户）
2. **看完后你希望他们知道什么？**（一张图应只传达1个核心信息）
3. **数据结构是什么？**（对比/趋势/分布/关系/组成）

> ❌ 错误：先画图，再想配什么数据
> ✅ 正确：先确定「我要说什么」，再找最能揭示这个洞见的图

---

### 原则 2：选对图类型（图类型选择矩阵）

```
你的核心问题 → 推荐图类型

比较大小          → 柱状图 / 条形图
看趋势            → 折线图 / 面积图
看分布            → 直方图 / 箱线图 / 小提琴图
看比例            → 饼图（≤5类）/ 堆叠柱状图（>5类）
看相关性          → 散点图 / 相关热力图
看流转            → 桑基图 / 漏斗图
看层级/结构       → 树形图 / 组织架构图
看过程            → 流程图 / 泳道图
看地理分布        → Choropleth / 气泡地图
展示抽象关系      → 概念图 / 框架图
展示机制          → 通路图 / 循环图
多维度综合        → 雷达图 / 平行坐标图
```

---

### 原则 3：视觉层级（Visual Hierarchy）

图中所有元素都应有明确的**阅读优先级**：

| 优先级 | 元素 | 设计策略 |
|--------|------|---------|
| 1级（2秒内抓住） | 核心洞见/标题 | 最大字/最深色/中心位置 |
| 2级（5秒内理解） | 主数据/主要图形 | 强调色，清晰可辨 |
| 3级（细读时获取） | 轴标签/图例/注释 | 小字，灰色，不抢主图 |
| 4级（按需查找） | 数据来源/方法说明 | 最小字，页脚 |

**核心原则**：读者的眼睛应该沿着你设计的路径移动，而不是四处游荡。

---

### 原则 4：Tufte 的数据墨水比

**数据墨水（Data-Ink）**：直接表示数据的墨水  
**非数据墨水（Chartjunk）**：装饰性的墨水，不携带信息

```
要删除的常见 Chartjunk：
✗ 3D效果（永远不要用3D柱状图、3D饼图）
✗ 多余的网格线（只保留必要的参考线）
✗ 边框（大多数图不需要边框）
✗ 过多的颜色（≤5种）
✗ 阴影效果
✗ 装饰性图标/背景图
✗ 重复的图例（能直接标在数据上则直接标）
```

---

### 原则 5：格式塔原则（Gestalt Principles）

| 格式塔法则 | 含义 | 设计应用 |
|-----------|------|---------|
| **邻近性（Proximity）** | 靠近的元素被感知为一组 | 相关数据/标签要靠近对应图形 |
| **相似性（Similarity）** | 相似的元素被感知为同类 | 同类数据用同颜色/形状 |
| **连续性（Continuity）** | 眼睛倾向于跟随连续线条 | 流程图的线条要流畅 |
| **封闭性（Closure）** | 脑补不完整的形状 | 可用不完整轮廓暗示容器 |
| **图底关系（Figure-Ground）** | 前景与背景的区分 | 核心元素要与背景有足够对比度 |
| **对称性（Symmetry）** | 对称结构感知为整体 | 对比图（左vs右）可用对称布局 |

---

### 原则 6：颜色的正确使用

#### 颜色的四种用途（不可混用）

| 用途 | 说明 | 配色推荐 |
|------|------|---------|
| **分类（Qualitative）** | 区分不同类别 | ColorBrewer Set1/Set2，≤8色 |
| **顺序（Sequential）** | 表示从低到高的连续量 | Blues / YlOrRd，单色渐变 |
| **双向（Diverging）** | 表示两个极端之间的变化 | RdBu / PuOr，白色为中心 |
| **强调（Highlight）** | 突出特定数据点 | 主图用灰色，强调用亮色 |

#### 颜色禁忌

```
❌ 红绿组合（色盲友好：约8%男性无法区分）
❌ 彩虹色谱用于连续数据（感知非线性）
❌ 超过7种颜色表示类别
❌ 淡色文字在白背景（对比度不足）
❌ 颜色与形状冗余但不一致
```

#### 推荐工具
- **ColorBrewer** (colorbrewer2.org)：按场景推荐配色方案
- **Okabe-Ito palette**：色盲友好的8色方案
- **Coolors.co**：配色方案生成

---

### 原则 7：文字与标注规范

| 元素 | 规范 |
|------|------|
| **图标题** | 简洁说明核心洞见（不只是"X vs Y"，而是"X在Y条件下显著更高"）|
| **轴标签** | 含单位，不用缩写（除非领域通用）|
| **直接标注（Direct Labeling）** | 优先在数据旁直接标注，而不是图例 |
| **注释（Annotation）** | 用箭头+文字指向关键特征，指导读者关注 |
| **图例（Legend）** | 放在靠近数据的位置，顺序与数据顺序一致 |
| **字体** | 无衬线字体（Arial/Helvetica），正文≥8pt，科研图≥5pt |
| **数字格式** | 避免过多小数位；大数用K/M/B简化 |

---

### 原则 8：揭示关系，而非堆砌观点

这是你提到的最核心问题：**图不应是观点的堆砌，而应是关系的揭示。**

#### 「观点堆砌」型图 vs 「关系揭示」型图的区别

| | 观点堆砌型 | 关系揭示型 |
|--|-----------|-----------|
| **外形** | 文字+图标列表，每条都独立 | 图形元素之间有视觉连接 |
| **传达** | 读者看到5个孤立要点 | 读者看到5个要点如何相互作用 |
| **记忆** | 需要记忆每个观点 | 结构本身帮助记忆 |
| **例子** | 5个并排的方框 | 循环图/因果图/层级图 |

#### 如何把观点变成关系

1. **找到观点之间的逻辑类型**：
   - 时间顺序 → 流程图 / 时间线
   - 因果关系 → 因果图 / 箭头注释
   - 包含关系 → 嵌套图 / 同心圆
   - 对立关系 → 双轴图 / 对比表
   - 正反馈 → 循环图
   - 多因一果 → 鱼骨图 / 汇聚箭头
   - 权衡关系 → 2×2矩阵
   - 层级关系 → 金字塔 / 树形图

2. **用「结构」而非「列表」组织内容**：
   - 把5个并排方块改成有层级、有连接的结构图
   - 加上代表关系的箭头，并标注箭头含义（"导致"/"属于"/"需要"）

3. **设计「视觉逻辑动线」**：
   - 读者的眼睛应该从左→右，或从上→下，沿着逻辑流动
   - 用颜色/大小/位置暗示重要性

---

### 原则 9：小倍数（Small Multiples）原则

当你需要对比多个相似数据集时，**不要把所有数据挤在一张图上，而是用统一格式的小图排列**。

- 同样的坐标轴范围
- 同样的图形类型
- 并排或网格排列
- 读者自然能对比差异

Tufte 称之为"世界上最好的设计解决方案之一"。适用：多国对比、多时间点、多条件实验。

---

### 原则 10：可访问性（Accessibility）

- 色盲友好配色（避免红绿）
- 图形+颜色双重编码（不只靠颜色区分）
- 足够的对比度（WCAG 2.1 Level AA）
- 屏幕阅读器：添加Alt text描述
- 打印版：确保黑白打印仍可辨认

## 第三部分：场景专项规范

### 场景 A：学术科研图（Research Paper Figures）

**核心原则**：精确、可重复、信息完整、无主观美化

```
必须包含：
✅ 坐标轴标签+单位
✅ 误差类型（SD/SEM/95%CI，必须说明）
✅ 样本量（n=）
✅ 统计显著性标注（*p<0.05, **p<0.01, ***p<0.001, ns）
✅ 比例尺（显微镜图）
✅ 图说（Figure caption）完整，看图说能独立理解图
✅ 色盲友好配色

禁止：
❌ 未说明误差类型的误差棒
❌ Y轴不从0开始（柱状图必须从0）
❌ 过度美化（图示软件不能用Photoshop改数据！）
❌ 缺乏统计检验说明
❌ 重要数据被截断
```

**分辨率要求（Nature/Science）**：
- 线图：≥600 dpi
- 含像素图的组合图：≥300 dpi
- 纯像素图（显微镜等）：≥300 dpi（推荐450）
- 字体：Arial/Helvetica，最小5pt

**推荐工具**：
- R + ggplot2（最强统计图，完全可定制）
- Python + matplotlib/seaborn/plotly
- Prism（医学生物领域标准）
- BioRender（生物机制图标准）
- Illustrator / Affinity Designer（最终排版）

### 场景 B：产品/技术文档图

**核心原则**：准确、易于开发者/PM理解、版本可维护

```
必须包含：
✅ 清晰的图层分离（核心/辅助/注释）
✅ 统一的形状语言（同类型用同形状）
✅ 箭头含义明确（触发/数据流/包含/继承）
✅ 版本号或日期
✅ 图例（如有多种元素）

流程图规范：
- 开始/结束：椭圆
- 过程：矩形
- 决策：菱形
- 数据：平行四边形

```

**推荐工具**：

- PlantUML（代码生成UML）
- Mermaid（Markdown内嵌图）

### 场景 C：商业演示/PPT图

**核心原则**：一目了然、支持口头讲解、视觉专业

```
PPT图的三条铁律：
1. 一张图只讲一个故事（删除所有"顺便"的内容）
2. 图的标题不是"图3"，而是洞见句子（"Q3收入超预期14%"）
3. 图要"自我解释"——演讲者不在时，图也能被理解

常见错误：
❌ 把Excel默认图直接复制粘贴（默认设置极差）
❌ Y轴不从0开始（夸大视觉差异）
❌ 过多颜色（每页≤3种主色）
❌ 文字太小（演示时观众看不见）
❌ 没有标注关键数据点（"顶峰是什么时候？"）
```

**咨询公司图表风格（McKinsey/BCG/Bain）**：
- 标题永远是观点句，不是描述句
- 图表配色极简（单色+强调色）
- 数字大字显示，轴标签小字
- 永远有数据来源标注

**推荐工具**：
- PowerPoint + Think-cell插件（咨询公司标配）
- Keynote（macOS，动效好）
- Canva（快速制作，模板丰富）
- Beautiful.ai（AI辅助布局）

### 场景 D：公众号/社交媒体推送图

**核心原则**：秒懂、视觉吸引、适配移动端

```
核心设计目标：
- 3秒内传达核心信息（用户滑动时的注意时窗）
- 适配手机屏幕（字体≥14px，重要元素不在边缘）
- 配色有记忆点（品牌色或主题色）

公众号图片规范：
- 封面图：900×383px（推荐），或1:1正方形
- 内文图：宽度≤900px，文件<2MB
- 推荐格式：JPG（照片类），PNG（含透明背景）

信息图设计技巧：
✅ 关键数字放大（数字比文字更快被感知）
✅ 图标辅助文字（降低理解成本）
✅ 颜色分区（每个信息块用不同底色）
✅ 白色/浅色文字在深色背景效果好
✅ 加入品牌LOGO（提高识别度）

禁忌：
❌ 纯文字列表配图（不如直接写文章）
❌ 字体太多（≤3种字体）
❌ 像素太低（放大后模糊）
❌ 信息密度太高（1张图≤5个信息点）
```

**推荐工具**：
- Canva（最易用，模板丰富）
- 创客贴/稿定设计（中文场景）
- Adobe Express
- Figma（设计师级别）

### 场景 E：教育/课程图示

**核心原则**：降低认知负荷，分层讲解，引导式理解

```
2. 渐进揭示（Progressive Disclosure）：
   - PPT：逐步出现，不要一次全显示

3. 类比图（Analogy Diagram）：

4. 前后对比（Before-After）：

避免：
❌ "地图恐惧症"——把所有内容塞进一张图（让人望而生畏）
❌ 专业术语没有解释

```

## 第四部分：案例分析——从「糟糕」到「优秀」

### 案例 1：展示用户增长数据

**糟糕版本**：
- 用3D饼图展示Q1-Q4用户占比
- 背景有装饰性渐变
- Y轴从50万开始（截断）
- 图标题写"用户数量统计"

**优秀版本**：
- 改为折线图（趋势比占比更重要）
- 白色背景，无装饰
- Y轴从0开始
- 在增长拐点处添加注释（"推出裂变功能"）
- 标题改为："用户数在Q3裂变功能上线后增速翻倍"

**改进点**：选对图类型（趋势→折线）、增加上下文注释、标题传达洞见。

---

### 案例 2：展示复杂的信号通路（科研）

**糟糕版本**：
- 10个方框用箭头随机连接
- 所有箭头相同颜色、相同粗细
- 没有图例
- 没有空白，元素拥挤

**优秀版本**：
- 激活箭头（→）用绿色/实线，抑制箭头（⊣）用红色/虚线
- 箭头粗细表示效应强度
- 分区域用浅色背景块区分"细胞质/细胞核/胞外"
- 关键蛋白用独特形状（激酶=齿轮图标）
- 标注实验证据来源
- 留足白空间

**改进点**：颜色编码关系类型、空间编码分区、双重编码（颜色+线型）。

---

### 案例 3：商业演示——展示产品策略

**糟糕版本**：
- 五个并排方框各写一个策略
- 方框之间无箭头，无关系
- 标题写："五大产品策略"
- 颜色用了5种不同颜色

**优秀版本**：
- 改为漏斗图或循环图，体现策略间的逻辑顺序/依赖关系
- 如果是循环：用箭头连接，标注每步的核心驱动力
- 强调色只用于"当前聚焦的策略"，其他用灰色
- 标题改为："从市场渗透到生态构建——三年增长路径"

**改进点**：用图形关系替代列表、减少颜色、标题传达逻辑。

---

### 案例 4：公众号推送——展示调研数据

**糟糕版本**：
- 把Excel表格截图放进文章
- 六组数据，每组用不同颜色
- 标题"2026年科研人员工具使用调查"

**优秀版本**：
- 只摘取最重要的1-2个发现制图
- 横向条形图+直接标注百分比
- 同类别统一颜色，强调最重要的一条用品牌色
- 标题："73%的科研人员每天使用AI，但只有12%觉得真正有用"
- 注释样本量和来源

**改进点**：聚焦核心发现、条形图替代表格、标题即洞见。

## 第六部分：绘图自查清单

### 绘图前
- [ ] 我知道这张图的目标读者是谁
- [ ] 我知道这张图的核心信息是什么（一句话能说清楚）
- [ ] 我选的图类型符合数据结构和表达目的
- [ ] 我已经考虑过其他可能的图类型

### 绘图中
- [ ] 图中每个元素都有存在的理由（没有纯装饰）
- [ ] 颜色使用有意义（不超过5种，色盲友好）
- [ ] 字体可读（无衬线，足够大）
- [ ] 轴标签含单位
- [ ] Y轴起点合理（柱状图必须从0）

### 绘图后
- [ ] 标题传达洞见，不只是描述（"X比Y高20%"而非"X vs Y比较"）
- [ ] 读者5秒内能抓住核心信息
- [ ] 关键数据点有注释
- [ ] 有数据来源/方法说明（科研必须，传播推荐）
- [ ] 在小屏幕（手机）上仍然可读
- [ ] 黑白打印仍能区分（科研必须）
