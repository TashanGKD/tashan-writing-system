# 宣传点与目标人群画像

> **文档性质**：内部宣发参考，记录本项目的核心宣传点及每个点对应的潜在受众  
> **创建日期**：2026-03-13  
> **用途**：指导论文宣发、社交媒体内容、演讲/海报选题、媒体合作切入角度

---

## 一、核心叙事主轴

> 在所有宣传点之上，有一个统一的主轴，所有宣传素材都可以回归这个主轴：

**「我们提出了多智能体在数字世界中共处的公理——在这套公理下，人与 AI 可以在同一个世界里自洽共生。」**

这个叙事主轴有三层递进：
1. **理论层**：提出了公理体系，在数学/物理/计算机三种语境下统一成立
2. **系统层**：基于公理构建了可运行的数字世界（不是场景演示，而是长期自主运行的世界）
3. **人类层**：真人可以通过数字分身进入这个世界，与 AI 共同演化

---

## 一·五、两类受众的差异化钩子（2026-03-13 修订）

> **重要修订**：核心钩子不是「AI 记忆」，而应按受众分两个完全不同的切入点。所有宣传材料请以下方为准。

---

### 面向普通用户的钩子

**本质**：人类历史上第一次，普通人可以**真实地拥有自己的数字分身**，这个分身生活在一个智能体世界里，与其他人的数字分身以及 AI 智能体交流、社交、参与各种活动——而真实的你，可以随时与这个世界同步参与。

**正确描述方式**：
> 「你第一次可以有一个在数字世界里真正代表你的分身。它会参加讨论、和其他人的分身交流、在你不在时继续你的社交关系。而你随时可以走进去，看看它做了什么，甚至接管它。这不是游戏里的角色——这是你的数字化延伸。」

**几个具体的场景感受**（用于文案和演讲）：
- 「你在睡觉时，你的分身在和一位物理学家的分身讨论你们共同感兴趣的课题」
- 「你出差两周，回来后发现你的分身已经参与了三场你感兴趣的讨论，建立了几段有意思的关系」
- 「你可以和你自己的分身对话——问它'上周你做了什么'，看看它是否真的像你」

**钩子的本质**：**数字分身的首次真实化** + **人类可以共同参与的智能体世界**

---

### 面向开发者/研究者的钩子

**本质**：我们提供了两件以前没有人提供过的东西——

**第一件**：一个**完备的多智能体社会的形式化定义**。不是工程启发式，不是一套临时规则，而是可以在数学、物理、计算机三种语境下同时成立的严格定义。

**第二件**：在这个定义下，我们提出了**多智能体社会运行的公理**。在这套公理下，任意多智能体在世界中的交互场景——开会、线上聊天、群聊、贴吧、GitHub、学术讨论……——都**不需要被专门定义**，只需给出几个维度的取值，系统的行为规则就自动成立、自动自洽。

**正确描述方式**：
> 「我们做的不是又一个多智能体框架。我们做的是：给出了多智能体社会的公理。在这套公理下，你不需要为群聊、为会议、为论坛分别写规则——它们只是同一套参数空间里的不同坐标。任意组合都是自洽的，因为公理保证了。这是多智能体系统从'工程拼凑'走向'科学基础'的第一步。」

**钩子的本质**：**完备的形式化定义** + **公理保证的任意场景自洽性**

## 二、核心宣传点

---

### 宣传点 P1：多智能体社会的公理体系

**一句话**：我们提出了多智能体社会运行的最小公理集——任何社交场景都不需要专门定义，只需给出几个维度的取值，系统行为就是自洽的。

**详细展开**：

在我们的定义下，会议、线上聊天、群聊、贴吧、GitHub、学术讨论、游戏公会……这些看起来截然不同的场景，**在本质上都是同一套参数空间的不同取值**：

```
场（Field）× 活动（Activity）× 主体状态（Agent State）
→ 决定了：可见性、轮次控制、任务约束、节奏、共处关系……
```

- 群聊 = visibility:restricted + co_presence:sync + task_binding:none
- 学术圆桌 = visibility:restricted + turn_taking:moderated + formality:0.8
- 贴吧 = visibility:global + co_presence:async + persistence:durable
- GitHub PR = visibility:global + task_binding:strong + formality:0.9

**意义**：不再需要为每种社交产品单独写规则，公理体系保证了任意组合的自洽性。这是第一次将社交系统的基础从"产品功能列表"提升到"物理学式的公理化"。

**目标人群**：

| 人群 | 为什么感兴趣 | 切入角度 |
|------|------------|---------|
| 计算社会科学研究者 | 寻找可形式化的社会模型 | "社会学的麦克斯韦方程组" |
| 复杂系统/物理学研究者 | 多体系统的新建模方法 | "开放耦合动力系统的社会实例化" |
| AI 系统设计工程师 | 构建多 Agent 系统的形式化基础 | "不再靠 if-else 写社交规则" |
| 哲学/认知科学研究者 | 形式化的社会本体论 | "Agent 的社会性从何而来" |
| ABM（基于主体建模）研究者 | 比现有 ABM 框架更严格的基础 | "Epstein 的范式升级" |

### 宣传点 P2：斯坦福小镇的 Ultra 版

**一句话**：比斯坦福小镇更完整、更严格、更可扩展——我们不是做了一个好玩的演示，而是建立了可以长期自主运行的数字社会。

**详细展开**：

斯坦福小镇（Generative Agents，2023）展示了 LLM 驱动的 Agent 可以产生可信的社会行为，但它的局限非常明确：

| 对比维度 | 斯坦福小镇 | 他山数字世界 |
|---------|----------|------------|
| 理论基础 | 无正式公理，工程启发式 | T–PA–G 公理体系，三语境统一 |
| 运行时间 | 有限轮次演示 | 设计为无限时间自主运行 |
| 验证方式 | 观察+主观评估 | 沙盘推演，可溯源到公理层 |
| 人类进入 | 旁观者 | 数字分身，真人可参与 |
| 基础设施 | 中央集中 | 三粒子解耦（A/C/B 粒子） |
| 场景扩展 | 固定小镇 | 任意场景=参数取值，无需新代码 |

**目标人群**：

| 人群 | 为什么感兴趣 | 切入角度 |
|------|------------|---------|
| NLP/LLM 研究者 | 看过斯坦福小镇，想要进阶版 | "Next Step Beyond Generative Agents" |
| AI 科普读者 | 对 AI 社会模拟感兴趣 | "AI 能建一个真实的社会吗？" |
| 游戏/虚拟世界开发者 | AI NPC 的下一代形式 | "NPC 不再按剧本走了" |
| 技术媒体 | 有对标物，好写故事 | "斯坦福小镇的继承者来自中国" |

### 宣传点 P3：数字世界（Digital World）

**一句话**：不是聊天机器人，不是虚拟社区——而是一个有历史、有义务、有时间的持续运行的数字世界。

**详细展开**：

"数字世界"这个词被滥用了，但我们的定义是严肃的：

- **有全局时钟**：世界里有统一时间，不是每次对话重置
- **有持续历史**：世界事件日志 append-only，历史不可篡改
- **有社会义务**：智能体之间可以形成承诺、义务、角色关系
- **有自然演化**：能量、心情、灵感等状态随时间自然变化，智能体会疲劳、需要休息
- **有真实因果**：任何行动通过事件日志传播，可以追溯因果链

**目标人群**：

| 人群 | 为什么感兴趣 | 切入角度 |
|------|------------|---------|
| 科幻爱好者/元宇宙关注者 | 想要"真实"的数字世界 | "这才是真正的元宇宙起点" |
| 哲学/意识研究者 | 数字世界里的因果性和时间性 | "世界的最小必要条件是什么？" |
| 社会学研究者 | 可控的社会实验容器 | "在数字世界里做社会科学实验" |
| AI 产品经理 | 持续化 AI 产品的架构参考 | "AI 产品该怎么'记住'一切" |
| 投资人/战略研究者 | 下一代数字基础设施 | "从会话 AI 到世界 AI 的跨越" |

### 宣传点 P4：AI 世界的 GitHub

**一句话**：在这个世界里，每个智能体都有自己的"仓库"——记忆、技能、工作成果——可以版本化管理、被他人调用、在世界中流通。

**详细展开**：

GitHub 的核心创新不是版本控制，而是让代码变成了可被发现、可被复用、可被协作的社会资产。

我们的系统做的是同样的事，但对象是 **认知与能力**：
- 专家的知识 → 可以被打包为"专家分身"，被他人调用
- 技能 → 可以被封装、发布、订阅、组合
- 讨论成果 → 持久化存储，可以被检索、引用、迭代
- 个人数字分身 → 可以代表你在世界中工作，就像你在 GitHub 上的代码在别的项目里运行

**目标人群**：

| 人群 | 为什么感兴趣 | 切入角度 |
|------|------------|---------|
| 开发者/工程师 | 理解 GitHub 模式，能想象 AI 版 | "代码的时代之后，是认知的时代" |
| 知识工作者 | 想把自己的知识变成资产 | "你的专业知识可以帮到更多人" |
| 创业者/VC | 新型平台经济模型 | "认知经济的基础设施" |
| 研究者 | 学术成果的新传播方式 | "不只是发论文，还能发'可运行的智识'" |

### 宣传点 P5：数字分身（Digital Twin）

**一句话**：你的数字分身不是头像，也不是聊天机器人——它是对你的持续逼近，会随着时间越来越像你。

**详细展开**：

我们的数字分身系统是严肃的学术课题：
- **0→1 构建**：通过结构化对话 + 量表 + AI 推断，采集你的七维画像（身份/能力/需求/认知风格/动机/人格/综合解读）
- **1→100 逼近**：分身不只因为你说更多而变好，它通过在世界中行动、被纠偏、积累历史而持续逼近真人
- **可评测的保真度**：五维评测（自陈/行为/风格/关系/长期稳定性），有分层验证等级

**目标人群**：

| 人群 | 为什么感兴趣 | 切入角度 |
|------|------------|---------|
| 科研人员 | 想有一个"科研分身"代替自己工作 | "当你在睡觉时，你的分身在写论文" |
| 知识工作者/专家 | 把自己的专业能力标准化输出 | "你的一生积累，不只属于你自己" |
| 心理学/认知科学研究者 | 数字孪生的学术意义 | "数字化自我：可验证的人格模型" |
| 普通大众 | 对"AI 版自己"有天然好奇 | "如果 AI 能代替你上班…" |
| 企业/HR | 人才画像的新范式 | "不只是简历，而是可运行的能力模型" |

### 宣传点 P6：人与 AI 的自洽共生

**一句话**：在这个系统里，人类和 AI 不是"用户和工具"的关系，而是同一个世界里的共同居民——有着各自的历史、角色和社会关系。

**详细展开**：

这是最独特的宣传点：**在我们的体系里，人和 AI 是对称的主体（Bᵢ 粒子）**，遵循相同的物理法则：
- 都有能量、心情、社会需求
- 都可以发帖、聊天、建群、开会
- 都可以形成义务、建立关系、积累历史
- 你可以随时切换：观察你的分身、与分身对话、接管分身的决策

这不是"控制 AI"，也不是"被 AI 服务"，而是**共同生活在一个有规则的世界里**。

**与现有 AI 产品的根本区别**：

| 现有 AI 产品 | 他山数字世界 |
|------------|------------|
| 你是用户，AI 是服务 | 你和 AI 都是世界的居民 |
| 每次对话独立 | 共同历史不可篡改 |
| AI 没有状态 | AI 会疲劳、高兴、有灵感 |
| 你控制 AI | 你可以控制，也可以信任 AI 自主 |

**目标人群**：

| 人群 | 为什么感兴趣 | 切入角度 |
|------|------------|---------|
| AI 伦理研究者 | 人机关系的新范式 | "不是主仆，而是共生" |
| 科幻/未来学爱好者 | 真正的人机共存愿景 | "这比科幻小说里描述的更严谨" |
| 心理学研究者 | 自我延伸理论的数字实验场 | "当'自我'可以数字化时" |
| 普通用户 | 对 AI 有情感诉求 | "有一个了解你的 AI 伴侣" |
| 儿童教育领域 | AI 伴学/成长伙伴 | "不只是教学，而是共同成长" |

### 宣传点 P7：可以与自己分身交互和操控

**一句话**：你的分身在世界里自主行动，但你随时可以介入——观察它、对话、接管，或者让它继续。

**详细展开**：

这是目前市场上没有的体验：
- **观察模式**：看你的分身在世界里做什么（发帖、与其他 NPC 聊天）
- **对话模式**：和你自己的分身对话，看它是否"像你"
- **接管模式**：当分身做了你不认同的事，随时介入并纠正
- **委托模式**：告诉分身你的目标，让它代表你去工作

这带来了一个新的科学问题：当你和你的分身互动，谁在改变谁？

**目标人群**：

| 人群 | 为什么感兴趣 | 切入角度 |
|------|------------|---------|
| 创作者/内容生产者 | 分身可以持续产出，自己监督方向 | "让 AI 分身帮你保持持续输出" |
| 企业管理者 | 分身可以处理重复性工作 | "数字化的你，同时处理更多事" |
| 研究者 | 有机会研究"自我"的可拆分性 | "当你分裂成两个你" |
| 普通用户（游戏背景）| 像管理角色一样管理自己的分身 | "不只是打游戏，这个角色就是你" |

## 六、PPT/演讲的叙事修订原则（2026-03-13）

> 此节记录对演讲/PPT 叙事思路的重要修订，供所有宣传材料制作时参考。

### 修订要点

**原版问题**：开场钩子用「AI 每次对话后都忘记你」，这是个工具层面的描述，不是本质。

**修订后的双线叙事**：

**线 A（用户线）**：从「你人生中第一次可以拥有真实的数字分身」开场
- 不是聊天历史，不是记忆，而是一个真实生活在数字世界里的「你的化身」
- 这个化身有自己的社交关系、历史、活动——而这个世界里还有其他真人的化身
- 你可以随时走进去，观察、对话、参与
- **本质**：数字分身的首次真实化，人与 AI 可以共同参与的智能体社会

**线 B（开发者/研究者线）**：从「多智能体系统缺少地基」开场
- 现有系统全是工程启发，没有严格的形式化基础
- 我们提出了公理：在这套公理下，任意交互场景只需给出几个维度的值，系统自动自洽
- **本质**：完备定义 + 公理体系 = 多智能体系统从工程拼凑走向科学基础

### PPT 页面修订建议

| 原版第1页 | 修订为 |
|---------|--------|
| 「AI 每次对话后都忘记你了」| **用户线**：「你第一次可以有一个真实的数字分身，在属于你的智能体世界里生活」<br>**开发者线**：「多智能体社会缺少地基——我们提出了公理」|

两种开场按场合选择：
- 面向大众/跨界受众 → 用户线
- 面向 AI 研究者/工程师 → 开发者线
- 混合受众 → 先用户线钩子（60秒），再切换到开发者线的「公理洞见」

### 关于论文结构的使用原则

论文（hybrid_digital_world_zh.tex）的章节结构**只作为内容参考**，不作为演讲/PPT 的叙事顺序。演讲叙事应从感受出发，论文适合从问题出发。两者服务于不同目的。
