## 宣发原则

1. **内容先于宣发**：宣发材料必须基于已有的真实内容（研究、代码、产品），不做超出实际进展的宣传
2. **分层宣发**：学术受众（论文/arXiv）→ 技术受众（GitHub/技术社区）→ 大众受众（科普文章）→ 商业受众（商业媒体）
3. **可追溯**：每篇发布的内容都要在本文件夹中存档，包括发布链接、发布时间、反馈数据
4. **中英双语**：面向国际学术/技术受众的宣发材料必须有英文版
